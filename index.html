Blog

This is my first year working on Web Dev, so far, this program has been really thrilling. Learning how much work it takes to create a website is intriguing. Looking back now, I feel as if the hard work of some websites is really underappreciated. But as my classes come to an end, I am required to finish some finals, and for my final in Web Dev, I've decided to do a research blog about CSS Sprites. So, what are they?

CSS Sprites are a type of technique in web development, that involves combining multiple images into a single image file to reduce the number of HTTP requests made by the browser. This method is commonly used to improve web page load times, especially when dealing with icons, buttons, or other small images. The history of sprites dates back all the way to 1975, the word sprites came from a technique in computer graphics, mostly used in video games. The idea was to have an image downloaded to its memory and show parts of the image. By doing that, it'll speed up the process of videos and clips. Before this, a program would have to continuously download new images which took a lot longer. This program at first was made for the increase in video games and they tried to level up their program game to give audiences a better game. A good example would be the older pixel PokÃ©mon games. The Sprite sheet consisted of over 1000 graphic objects. Even in the present sprites are still used in every website. It's the simple buttons and icons no one really pays attention to, like the buy button on Amazon, or even the logos people keep.

Even in our present-day format, CSS Sprites are still used but it's also evolved. The benefits of using them now are a reduced number of HTTP requests, reduced overall image size, independence of JavaScript, reduced loading time, and Flexibility. However, even with all those pros, everything comes with cons as well. The Demerits of CSS Sprites are higher development time, more maintenance time, and less SEO friendly. Seo friendly just means it's easier for search engines to crawl and understand the structure and content of a website.

So how do you actually use CSS Sprites? It seems to be somewhat "simple." You insert images and by positioning them and moving them in certain areas, you're able to show certain images that's already downloaded and don't have to download new images which can take a lot longer. By using background-repeat: no-repeat; you're able to show only what you want to show with positioning. Most of the positioning is controlled by the command (width: and height:) Other commands used are (background-image and background-position) A good example would be

.flags-canada { height: 128px; background-position: -5px -5px;}
